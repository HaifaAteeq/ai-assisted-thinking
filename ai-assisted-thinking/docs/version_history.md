# Version History
## AI-Assisted Thinking Protocol

---

## v2.2 (Current) - January 2026

### Added
- **Recovery Mode** - Handles brain freeze and cognitive overload
- **Topic Decomposition** - Breaks broad topics into micro-concepts
- **Language Support** - Flexibility for second-language learners
- **Fatigue Detection** - Monitors and responds to cognitive fatigue
- **Energy Check** - Adjusts session pace based on user state

### Changed
- Cognitive load management added as foundational principle
- Session start now includes energy/time/language check

### Why These Changes
Based on real testing: Learning math for ML as a second-language learner revealed that the protocol needed to handle:
- Brain freeze (couldn't be addressed by pushing harder)
- Broad topics (caused paralysis)
- Language fatigue (additional cognitive load)

---

## v2.1 - January 2026

### Added
- **Overconfidence Detection** - Verifies claimed expertise before accepting
- **Retention Decision Tree** - 80%/40%/<40% thresholds for continuity
- **Session End Criteria** - Clear rules for when to end sessions
- **System Limitations** - Explicit scope boundaries
- **Meta-Tracking** - System improvement feedback
- **Progress Tracking Template** - Detailed session logging

### Changed
- Foundation Mode reframed as "Grounding Context" (not explanation)
- Core intention clarified (self-deception vs dishonesty)
- PhD implementation phased (not all modes at once)

### Why These Changes
Critical feedback identified:
- Foundation Mode contradicted "never explain" rule
- Curriculum Mode too complex for initial PhD
- Missing handling for overconfident users
- No clear session boundaries

---

## v2.0 - December 2025

### Added
- **Foundation Mode** - For users with zero background
- **Exploration Mode** - For aspiration-driven learners (no experience)
- **Curriculum Mode** - Build complete learning paths
- **Application Gap Mode** - For "read but can't apply" problem
- **Continuity Mode** - Cross-session retention and building
- **Six-mode architecture** - Covers all learner entry points

### Changed
- Expanded from single protocol to multi-mode system
- Added verification gates at multiple points
- Detection criteria more comprehensive

### Why These Changes
Testing revealed original protocol was too rigid:
- Complete beginners couldn't engage
- Aspiration-driven learners had no anchor
- "Read but can't apply" was most common complaint
- No handling for returning users

---

## v1.0 - December 2025

### Initial Release
- **5-Step Direct Protocol**
  1. Intent + Context Declaration
  2. Situate in Concrete Situation
  3. Articulation Through Application
  4. Reflective Challenge
  5. Reflection Log
- **Core Rules**
  - Never give direct explanations
  - Always verify through demonstration
  - Questions before answers
  - Block progression until verified
- **Detection Criteria**
  - Shallow vs deep understanding signals
  - Listing vs relating distinction

### Foundation
- Core intention: "This AI does not let me lie to myself about understanding"
- Based on learning science: productive struggle, retrieval practice
- Designed for self-directed learners

## Version Naming Convention

```
v[Major].[Minor]

Major: Significant architectural changes
Minor: Feature additions and refinements

Examples:
- v1.0 → v2.0: Single protocol → Multi-mode system
- v2.0 → v2.1: Added overconfidence detection, refined modes
- v2.1 → v2.2: Added recovery mode, language support
```

---

## Change Log Format

For each version:
1. **Added** - New features
2. **Changed** - Modifications to existing features
3. **Removed** - Deprecated features (if any)
4. **Why** - Reasoning behind changes (important for research documentation)
